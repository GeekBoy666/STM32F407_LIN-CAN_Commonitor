通信格式保持 ：波特率38400， “N，8，1”方式不变。

1. SET

CANOE发：SET01HI<CR><LF>
ETA回复   :  SETCH01==HIGH<CR><LF>
CANOE发：SET01L<CR><LF>
ETA回复   :  SETCH01==LOW<CR><LF>

指令执行结果是硬件GPIO的01通道输出高电平
其中通道号选值范围：00-31,共计32通道，可以驱动32继电器


2. GET

CANOE发：GET01回车
ETA回复   :  CH01==XXXX回车

数值01代表ADC的01通道，取值范围：00-15， 共计16通道。
命令执行结果是硬件GPIO（同时也是ADC转换能力）上的电压值，被转换成ADC值，ASCII码表示的。
即0-3.3V的电压线性对齐到0-4095， 即2的12次方。
然后CANOE依照硬件工程师指导来确定该ADC结果的用途。
比如：用于判定某个引脚是高电平还是低电平，或者用于读回某个引脚的具体电压值，电流值等，具体换算关系由硬件工程师提供。

*指令扩展
为了方便维护，保留一个调试功能的指令

CANOE发送： HELP回车
ETA将回硬件版本号，软件版本号，以及指令示例等

ETA收到的指令和应答情况会在液晶屏上显示出来。
